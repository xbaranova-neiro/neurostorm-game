# Нейрошторм — MVP (мобильный web)

Вертикальная мини-игра: **~70 с** раунда, **3 шкалы** (деньги / время / энергия), объекты (польза / ловушки / бустеры), **развилки**, **4 архетипа** финала + запасной, универсальный и персонализированный CTA. В HUD — условные **8 ч «дня»** + реальный отсчёт.

## Запуск локально

Из папки `neurostorm`:

```bash
cd neurostorm
python3 -m http.server 8080
```

Откройте в браузере: **`http://localhost:8080`** (четыре цифры: **8080**, не `808`).  
Для мобильного теста — с телефона в той же Wi‑Fi сети: `http://<ваш-IP>:8080`.

**Сервер не отвечает?** Сначала в терминале из папки `neurostorm` выполните `python3 -m http.server 8080` и оставьте окно открытым — пока команда работает, сайт доступен. Остановить — `Ctrl+C`.

**Важно:** в `index.html` подключён **`js/bundle.js`** (сборка без ES‑модулей). Если меняете файлы в `js/*.js`, пересоберите бандл:

```bash
cd neurostorm
npm install   # один раз: esbuild
npm run build
```

Для работы без `npm` можно вызвать `npx esbuild js/main.js --bundle --outfile=js/bundle.js --format=iife --target=es2020`.

## Управление

- **Тап** по зелёным/фиолетовым карточкам — поймать пользу или бустер.
- **Свайп в сторону** по красной ловушке — сбросить; если уйдёт вниз или тапнуть — штраф.
- **Развилки** — модальное окно, игра на паузе; у части сценариев вторая развилка зависит от первого ответа.
- **Перегрев корзины** — много зелёного подряд без паузы (секунды) бьёт по энергии.
- **Усталость** — при энергии &lt; 20 корзина двигается медленнее (свайп и стрелки); хаос-режим по спавну как раньше.
- **Экран результата** — «упущенный потенциал»: **100%** номинала по промахам/сгоревшим/ловушкам, **перегруз** (много выгод на поле — новые «задачи» бьют по темпу), **долгая пауза на развилке** (рынок не ждёт), **перегрев корзины** (слишком много пойманного подряд), плюс **ужесточение таймера** горящих карточек при очереди. Сумма на экране **анимируется рывками** + лёгкий «удар» по строке.

## Аналитика

События пишутся в консоль (`[analytics] ...`) и дублируются событием `neurostorm_analytics` на `window`. Подключите свой трекер в `js/analytics.js`.

События: `game_start`, `role_selected`, `wave_completed`, `booster_caught`, `trap_hit`, `trap_evaded` (красная ушла вниз мимо, без UI), `decision_choice`, `final_archetype`, `cta_click`, `cta_form_new_tab`.

## Структура

- `index.html` — экраны
- `css/main.css` — стили, контейнер ~9:16
- `js/data.js` — **контент**: роли (питч + история), объекты с **хуками** (подпись смысла), развилки (**контекст + подсказки** к вариантам), архетипы (боль, инсайт, ошибки, кейс, шаг, CTA)
- `js/narrative.js` — **голос рынка**: тикер в HUD, темы волн, варианты хаоса, оттенки комбо
- `js/game.js` — движок
- `js/diagnostics.js` — выбор архетипа
- `js/main.js` — навигация
- `js/analytics.js` — заглушка трекинга

## GetCourse и GitHub Pages

На хостинге **`*.github.io`** зелёная CTA **сразу открывает форму в новой вкладке** (встроенный виджет в модалке у GetCourse часто не работает с чужих доменов). На своём домене по-прежнему используется модалка.

Чтобы модалка работала и с GitHub Pages, в настройках школы GetCourse нужно разрешить домен вида `ваш-логин.github.io` (если такая опция есть).
